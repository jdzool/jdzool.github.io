Code: Discrete Event Simulation

Discrete Event Simulation (DES) is one way to model systems which have defined (discrete) state changes over time. This type of model is useful to understand constraints within a system caused by different processes and queuing behaviours. Common use cases are in logistics (for example package delivery, hospital optimisation) and network simulation (for example capacity management). In each case systems can occupy a set of defined states, and state are modelled to interact under defined business logic coded into the model. Running simulations of this kind can result in a better understand of capacity given demand in normally operation and under system stress scenarios.

The model created here: https://github.com/jdzool/agile-team-model is a discrete event simulation of an agile data team. SimPy is used as a modelling framework, skopt used for optimsation and matplotlib used for plotting outputs. This model simulates at a basic level the movement of tasks through various teams on a scrum board. Tasks are simulated as discrete events (http://simpy.readthedocs.io/), which consume resources from teams as they move through a defined software engineering process.

The outputs of this model are learnings related to:

  • The optimal size and shape of a team, given a work rate for each subteam
  • Practical understanding and framing of constraint type problems, covering topics similar to those described in Eli Goldratt’s The Goal.
